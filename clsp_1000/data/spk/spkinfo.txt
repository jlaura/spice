General Information About SPICE SP-Kernels
==========================================

   The purpose of the SP-kernel -- which stands for S(pacecraft) and
   P(lanet) Kernel -- is to allow ephemerides for any collection of
   solar system bodies, spacecraft or any other objects in 3D space to
   be combined under a common file format, and accessed by a common set
   of subroutines.

   The SPK files can be accessed and manipulated using the SPICE
   Toolkit APIs, calls to which can be integrated into user application
   programs. The information regarding how to obtain the SPICE system
   toolkit for a number of languages and computer platforms is located
   in the ``/software'' subdirectory of the main directory of this data
   set.

   To use SP-kernels, a Leapseconds (LSK) is required. Also, to use
   SP-Kernels containing positions given relative to the non-inertial
   and project-defined frames, a Frame Definitions (FK) and Planetary
   Constants (PCK) files are required.

   Refer to the ``spk.req'' document provided with the toolkit to get
   more information on how to access SP-kernel data.


Clementine SP-kernels Provided in This Data Set
===============================================

   The following Clementine SP-kernels are provided in this data set
   (only name portion of the file name is shown; the extension of all
   file names is ``.bsp''):

   Clementine Spacecraft Trajectory SPK files:

      clem_nrl    SPK file covering the whole mission, containing the
                  trajectory solution by the Spacecraft Operations
                  Team, Naval Research Laboratory. This solution was
                  done over the course of the mission operations and
                  was slightly updated by NAIF to eliminate the
                  erroneous data at the segment boundaries at the time
                  of archiving. The uncorrected version of this file
                  was called ``clemdef.bsp''.
 
      clem_gsfc   SPK file covering the whole mission, containing the
                  trajectory solution by the Clementine Gravity Team,
                  Goddard Space Flight Center. This solution was done
                  after the end of the mission and was labeled ``v002''.
                  The original name of this file was
                  ``SPKMERGE_9402191254_9405040324_CLEM.v002.bsp''.
 
      clem_jpl    SPK file covering the whole mission, containing the
                  trajectory solution by the Solar System Dynamics
                  group, Jet Propulsion Laboratory. This solution was
                  done in 1992. The original name of this file was
                  ``clem_ask020625.bsp''.

   Solar System Bodies Ephemeris data SPK file:

      de414       SPK file containing JPL's Planetary Ephemeris DE414,
                  the latest version at the time when this archive was
                  created.

   The difference of the trajectories provided by the three spacecraft
   SPK files varies greatly throughout the mission, ranging from a few
   meters for some orbits to hundreds of meters to kilometers for the
   fits near the propulsive maneuvers. Appendix 1 of this document
   provides a summary of the down-track, radial, and out-of-plane
   components of the maximum difference for each pair of SPKs for each
   orbit.


Kernel File Details
===================

   The most detailed description of the data in each SPK file is
   provided in metadata included inside the comment area of the file.
   This information can be viewed using utility programs COMMNT and
   SPACIT included into the NAIF Toolkit.

Appendix 1: Spacecraft Trajectory SPK Difference Summary Table
==============================================================

   This appendix provides a summary table of the down-track, radial and
   out-of-plane components of the maximum difference for each pair of
   spacecraft SPKs for each orbit.

   The table consists of 10 columns. The first column is the orbit
   number as defined by the ``clem.orb'' provided in the ``extras/orbnum''
   directory of this archive. The next three columns (2..4) contain
   view coordinate frame components -- ``down track'', ``in plane'' and
   ``normal to plane'' -- of the difference of the specified type
   between the first and second SPK files listed in the header. The
   next three columns (5..7) -- contain data for the difference between
   the first and third SPK files. And the last three columns (8-10) --
   contain data for the difference between the second and third SPK
   files. The units are kilometers.

   The difference components in the view coordinate frame were computed
   for each orbit using the following algorithm:

      -- A thousand states of Clementine with respect to the Moon in
         the J2000 frame were computed for each orbit from each pair of
         SPKs being compared. These states were evenly distributed
         between orbit start and stop times as specified by the orbit
         number file.

      -- For each pair of states, a position difference vector was
         computed by subtracting the state computed from the second
         file from the state computed from the first file. Then, a
         frame transformation matrix rotating these difference vectors
         from J2000 to the view coordinate frame defined by the state
         obtained from the first file was computed, and the position
         difference vector was rotated to the view frame coordinates
         using this matrix.

      -- Then, the difference components for the state with maximum
         relative difference were found and written to the table.

   For the orbits for which one of the SPK files that were compared
   didn't have complete coverage the differences were not computed and
   the corresponding positions in the table were left blank.
